# Product Brief: java

## Executive Summary

**java** 是一个专为5-6人小团队设计的轻量级内容管理平台，致力于让内部知识沉淀和分享变得简单高效。

对于内容创作者（作者），我们提供极简的后台编辑体验，让写作和发布流程行云流水；对于内容消费者（读者），我们提供干净、快速的阅读界面，让信息获取零 friction。

在当前市场上，Confluence、Notion等工具功能臃肿、学习成本高，而简单的Markdown方案又对非技术成员不友好。java 填补了这一空白——足够简单让所有人都能上手，又足够专业满足团队知识管理需求。

---

## Core Vision

### Problem Statement

5-6人的小团队在日常协作中面临信息分散、知识难以沉淀的困境：

- **写作门槛高**：现有CMS或文档工具界面复杂，功能堆砌让创作者难以专注
- **发布流程繁琐**：从写好内容到同事能看到，往往需要多个步骤和配置
- **阅读体验差**：现有方案要么太重（加载慢、干扰多），要么太简陋（不支持格式、搜索困难）
- **工具选择困境**：企业级方案（Confluence、语雀）功能过剩且成本高，轻量方案（Notion、飞书）又缺乏清晰的"发布"概念

### Problem Impact

当信息沉淀和分享不畅时，小团队会面临：

- **知识流失**：项目经验、技术方案随人员流动而丢失
- **重复造轮子**：同样的问题被多次解决，因为没有统一的知识库
- **新人融入慢**：新成员难以快速了解团队历史和技术积累
- **协作效率低**：信息散落在聊天记录、邮件、个人笔记中，查找困难

### Why Existing Solutions Fall Short

| 现有方案 | 主要问题 |
|---------|---------|
| **Confluence/语雀** | 功能臃肿、加载慢、学习成本高，小团队用不起来 |
| **Notion/飞书文档** | 过于灵活导致混乱，缺乏清晰的"发布"流程，阅读时易被消息干扰 |
| **Git + Markdown** | 技术门槛高，非技术成员无法参与 |
| **自建博客系统** | 部署维护成本高，写个文章要先配环境 |

### Proposed Solution

**java** 提供"简单到刚刚好"的内容管理体验：

**对作者：**
- 极简后台编辑界面，打开即写
- Markdown支持 + 实时预览
- 一键发布，无需额外配置
- 自动保存，不用担心内容丢失

**对读者：**
- 干净清爽的阅读界面，专注内容本身
- 按时间/分类浏览，快速发现内容
- 快速加载，无广告干扰
- 搜索历史文章，知识触手可及

### Key Differentiators

1. **极简主义设计**：没有功能膨胀，每个功能都为"轻松创作/阅读"服务
2. **清晰的发布边界**：区别于协作文档的"永远草稿"模式，强调"发布"的仪式感
3. **双边体验优化**：同时优化创作端和消费端，而非只关注一端
4. **小团队专属**：不为大企业复杂需求设计，专注5-6人团队的实际场景

## Target Users

### Primary Users

#### 内容创作者（作者）

**代表人物：小明**
- **角色**：团队技术骨干，负责写技术方案、项目总结、最佳实践
- **痛点**：现有工具要么太复杂（Confluence），要么缺乏发布概念（Notion）
- **需求**：极简编辑界面、Markdown支持、一键发布、自动保存
- **成功标准**：写的内容能被同事快速找到和阅读

**特征**：
- 有一定技术背景，习惯使用Markdown
- 愿意分享知识，但希望流程简单高效
- 写作频率：有项目或解决问题时记录

#### 内容读者

**代表人物：小红**
- **角色**：团队成员（包括新人），需要获取团队知识
- **痛点**：信息散落在聊天记录中，找不到历史资料
- **需求**：干净的阅读界面、快速搜索、按分类浏览
- **成功标准**：能快速找到需要的知识，了解项目背景

**特征**：
- 技术水平不一（从新人到资深）
- 主要消费内容，偶尔可能成为作者
- 使用场景：遇到问题查资料、新人了解团队历史

### Secondary Users

**系统管理员**
- 通常是团队技术负责人或项目负责人
- 负责系统部署和基础配置
- 需求：部署简单、维护成本低、数据安全

### User Journey

#### 作者旅程：发布一篇技术文章

1. **发现需求**：解决了一个技术难题，决定记录下来分享
2. **打开后台**：访问管理后台，点击"新建文章"
3. **创作内容**：在极简编辑器中撰写Markdown内容
4. **实时预览**：随时查看渲染效果
5. **一键发布**：选择分类，点击发布，文章立即可见
6. **获得反馈**：同事阅读后可能评论或私聊讨论

#### 读者旅程：查找项目资料

1. **产生需求**：遇到不熟悉的技术或项目，需要了解背景
2. **访问前台**：打开内容平台首页
3. **发现内容**：浏览最新文章或按分类查找
4. **阅读学习**：在干净的阅读界面中获取知识
5. **后续行动**：收藏重要文章或询问作者细节

## Success Metrics

### User Success Metrics

**核心成功指标：有用**

- **内容产出**：团队成员愿意持续发布文章（每周至少1篇新内容）
- **内容消费**：团队成员主动访问阅读（问题出现时首选查阅平台而非问人）
- **用户满意度**：团队成员反馈"比之前的工具好用"
- **零阻力使用**：新人无需培训即可自主发布和阅读内容

**质量指标：没有bug**

- **系统稳定性**：核心功能（编辑、发布、阅读）100%可用
- **数据安全**：无内容丢失事故
- **性能标准**：页面加载 < 2秒，编辑响应流畅

### Business Objectives

**内部工具价值验证**

- **时间节省**：减少重复解释和问题询问的时间（目标：每周节省2-3小时）
- **知识沉淀**：核心项目/技术文档从个人笔记迁移到共享平台（目标：80%核心知识可查）
- **新人上手**：新成员通过阅读文档独立理解项目（目标：1周内可自主工作）

### Key Performance Indicators

| 指标 | 目标值 | 时间框架 |
|------|--------|----------|
| 活跃作者比例 | 80%团队成员每月至少发布1篇文章 | 上线后1个月 |
| 平均每月新内容 | 4篇以上 | 持续 |
| 系统可用性 | 99.9% | 持续 |
| 用户主动使用 | 遇到问题首先查平台而非问人 | 上线后2个月 |

**成功定义总结**：团队觉得"有用"，系统"没有bug"——简单直接，专注核心价值。

## MVP Scope

### Core Features

#### 作者端（后台管理）

| 功能模块 | 具体功能 | 优先级 |
|---------|---------|--------|
| **用户认证** | 登录/登出 | P0 |
| **文章管理** | 文章列表、新建、编辑、删除 | P0 |
| **编辑器** | Markdown编辑 + 实时预览 | P0 |
| **分类管理** | 增删改查一级分类 | P0 |
| **发布流程** | 草稿/发布状态切换 | P0 |
| **自动保存** | 编辑时自动保存草稿 | P1 |

#### 读者端（前台展示）

| 功能模块 | 具体功能 | 优先级 |
|---------|---------|--------|
| **首页** | 文章列表（按时间倒序） | P0 |
| **分类浏览** | 按分类筛选文章 | P0 |
| **文章详情** | Markdown渲染为HTML展示 | P0 |
| **搜索** | 基础文章搜索 | P1 |

#### 基础设施

| 功能模块 | 具体功能 | 优先级 |
|---------|---------|--------|
| **数据模型** | 文章、分类、用户表设计 | P0 |
| **权限控制** | 区分作者（可写）和读者（只读） | P0 |

### Out of Scope for MVP

以下功能明确不在MVP范围内，将在后续版本考虑：

| 功能 | 推迟原因 | 计划版本 |
|------|---------|---------|
| 评论系统 | 增加复杂度，先验证核心流程 | V1.1 |
| 文章标签 | 一级分类足够MVP使用 | V1.1 |
| 用户注册 | 后台手动添加足够小团队使用 | V1.1 |
| 富文本编辑器 | Markdown足够技术团队使用 | V1.2 |
| 图片上传 | 先用图床链接，降低存储复杂度 | V1.1 |
| 数据统计/分析 | 先人工观察使用反馈 | V1.2 |
| 移动端适配 | 先专注桌面端体验 | V1.2 |
| 多语言支持 | 内部团队单语言足够 | V2.0 |

### MVP Success Criteria

MVP成功的标准：

1. **功能完整性**：上述P0功能全部可用，无明显bug
2. **用户采用**：80%团队成员能在1周内独立发布至少1篇文章
3. **体验验证**：团队成员反馈"比之前的工具好用"
4. **稳定性**：核心功能可用性达到99.9%

### Future Vision

**V1.1阶段（发布后1-2个月）：**
- 添加评论系统促进团队互动
- 文章标签支持多维度分类
- 图片上传功能
- 基础数据统计（文章阅读量）

**V1.2阶段（发布后3-6个月）：**
- 移动端响应式适配
- 写作数据分析（字数、时长统计）
- 文章收藏功能
- 搜索优化（全文搜索）

**长期愿景（V2.0+）：**
- 多团队/多工作空间支持
- 版本历史与协作编辑
- AI辅助功能（内容建议、SEO优化）
- 开放API支持第三方集成
