# UX Design Specification java

**Author:** Jxin
**Date:** 2026-02-17

---

## Executive Summary

### Project Vision

**java** 是一个专为5-6人小团队设计的轻量级内容管理平台（CMS），核心价值主张是**简单便捷**——让作者轻松发表，让读者轻松阅读。

产品定位是填补 Confluence（太复杂）和 Notion（缺乏发布概念）之间的空白，为小团队提供"刚刚好"的内容管理体验。

### Target Users

| 角色 | 代表人物 | 核心需求 | 技术水平 |
|------|---------|---------|---------|
| **作者** | 小明（技术骨干） | 极简编辑、Markdown支持、一键发布、自动保存 | 中-高 |
| **读者** | 小红（新人/成员） | 干净阅读界面、快速搜索、分类浏览 | 低-中 |
| **管理员** | 技术负责人 | 简单部署、用户管理、系统维护 | 高 |

### Key Design Challenges

1. **"简单便捷"的平衡** - 如何在功能完整性和极简界面之间找到平衡？
2. **Markdown编辑体验** - 如何让技术/非技术用户都能流畅使用？
3. **双边体验优化** - 如何同时优化作者端（创作）和读者端（阅读）？

### Design Opportunities

1. **沉浸式创作体验** - 创造专注的创作环境，减少干扰
2. **零摩擦发布** - 一键发布流程，减少操作步骤
3. **干净的阅读界面** - 极简美学，专注内容本身

## Core User Experience

### Defining Experience

**核心行动：作者端的"Markdown写作 → 一键发布"流程**

这是产品价值的核心——如果这个体验不流畅，作者就不会持续使用系统。

| 用户 | 核心行动 | 频率 |
|------|---------|------|
| **作者（小明）** | 写文章 → 一键发布 | 每周1-2次 |
| **读者（小红）** | 搜索/浏览 → 阅读文章 | 每日多次 |

### Platform Strategy

- **平台**：Web应用（前后台分离架构）
- **交互方式**：鼠标/键盘为主
- **MVP阶段**：桌面端优先，暂不考虑移动端适配
- **离线功能**：不需要

### Effortless Interactions

| 交互 | 期望体验 |
|------|---------|
| **发布流程** | 从"写完"到"发布"一键操作 |
| **自动保存** | 系统自动保存，用户无需手动操作 |
| **实时预览** | 编辑时实时看到渲染效果 |
| **内容发现** | 打开首页就能看到最新内容 |

### Critical Success Moments

- **作者**：点击"发布"后，文章立即对团队可见
- **读者**：5分钟内找到需要的资料，无需问3个人
- **新人**：无需培训就能完成第一次发布

### Experience Principles

1. **专注创作** - 编辑器界面极简，没有干扰元素
2. **零摩擦发布** - 从草稿到发布只需一次点击
3. **即时可见** - 发布后内容立即可见，无需等待
4. **干净阅读** - 阅读界面专注内容，没有广告/弹窗

## Desired Emotional Response

### Primary Emotional Goals

| 情感 | 描述 |
|------|------|
| **轻松** | "这个工具不复杂，我马上就能用" |
| **专注** | "我可以专心写内容，没有干扰" |
| **成就感** | "我写的东西被同事看到了" |
| **信任** | "我的内容不会丢失" |

### Emotional Journey Mapping

| 阶段 | 用户感受 | 设计支持 |
|------|---------|---------|
| **初次接触** | 好奇、期待 | 简洁的首页，清晰的价值传达 |
| **开始写作** | 专注、流畅 | 极简编辑器，实时预览 |
| **发布时刻** | 成就感、满足 | 一键发布，即时确认 |
| **发现内容** | 惊喜、收获 | 干净的阅读界面，快速搜索 |
| **持续使用** | 信任、依赖 | 稳定可靠，自动保存 |

### Emotional Design Principles

| 情感目标 | 设计选择 |
|---------|---------|
| 轻松 | 大按钮、清晰的文字标签、减少步骤 |
| 专注 | 极简界面、隐藏非必要功能、沉浸式编辑器 |
| 成就感 | 发布成功反馈、文章可见确认 |
| 信任 | 自动保存提示、稳定可靠的系统 |

## UX Pattern Analysis & Inspiration

### Inspiring Products Analysis

| 产品 | 优点 | 可借鉴的模式 |
|------|------|-------------|
| **Ghost** | 极简编辑器、干净阅读体验 | Markdown编辑、专注内容 |
| **Medium** | 沉浸式阅读、优雅排版 | 排版美学、阅读体验 |
| **Notion** | 实时预览、块编辑 | 编辑体验（避免功能过载） |
| **GitHub Docs** | 技术文档风格、代码高亮 | 技术内容展示 |

### Transferable UX Patterns

**导航模式：**
- 简洁的顶部导航（首页、分类、搜索）
- 后台管理侧边栏（文章、分类、用户）

**编辑模式：**
- 左右分栏：编辑区 + 实时预览区
- 顶部工具栏：基础格式化按钮
- 底部操作栏：保存草稿、发布按钮

**阅读模式：**
- 单栏居中布局，最大阅读舒适度
- 清晰的标题层级
- 代码块语法高亮

### Anti-Patterns to Avoid

| 反模式 | 问题 |
|--------|------|
| 功能堆砌 | 界面复杂，用户困惑 |
| 多级菜单 | 增加认知负担 |
| 复杂的富文本编辑器 | 学习成本高，易出错 |
| 过度动画 | 分散注意力，影响性能 |

## Design System Foundation

### Design System Choice

**推荐方案：Ant Design**

| 因素 | 分析 |
|------|------|
| **快速开发** | 完善的组件库，开箱即用 |
| **后台友好** | 专为管理后台设计，表格、表单组件丰富 |
| **中文生态** | 完善的中文文档和社区 |
| **可定制性** | 支持主题定制 |
| **成熟稳定** | 企业级应用验证 |

### Implementation Approach

**前台展示端：**
- 自定义样式，追求干净的阅读体验
- 借鉴Ghost/Medium的排版美学

**后台管理端：**
- 使用Ant Design组件库
- 快速构建CRUD功能

## User Journey Flows

### Journey 1: 作者发布文章

```
登录后台 → 点击"新建文章" → 编辑内容(左Markdown/右预览) → 选择分类 → 点击"发布" → 确认发布 → 文章可见
```

**关键节点：**
- 自动保存：每30秒自动保存草稿
- 实时预览：编辑时同步显示渲染效果
- 一键发布：无需额外配置，点击即发布

### Journey 2: 读者阅读文章

```
打开首页 → 浏览文章列表/按分类筛选/搜索 → 点击文章 → 阅读内容
```

**关键节点：**
- 首页即时展示最新文章
- 分类筛选快速定位
- 搜索支持标题关键词

### Journey 3: 管理员管理用户

```
登录后台 → 点击"用户管理" → 创建用户 → 设置角色 → 保存
```

## UX Consistency Patterns

### Button Hierarchy

| 类型 | 样式 | 用途 |
|------|------|------|
| **Primary** | 蓝色填充 | 主要操作（发布、保存） |
| **Default** | 白色边框 | 次要操作（取消） |
| **Danger** | 红色填充 | 危险操作（删除） |

### Feedback Patterns

| 类型 | 表现 |
|------|------|
| **成功** | 绿色Toast提示，2秒自动消失 |
| **错误** | 红色Toast提示，需手动关闭 |
| **加载** | Spin组件 + 骨架屏 |

### Form Patterns

- 必填项标红星
- 实时验证，错误提示在输入框下方
- 提交按钮在表单右下角

## Responsive Design & Accessibility

### Responsive Strategy

**MVP阶段：桌面端优先**

- 最小支持宽度：1024px
- 暂不实现移动端适配

### Accessibility (WCAG AA)

- 色彩对比度 ≥ 4.5:1
- 所有交互元素支持键盘操作
- 表单元素有明确的label
- 图片有alt属性

## Design Direction Decision

### Chosen Direction

**极简清爽风格**

- 大量留白，内容居中
- 清晰的视觉层级
- 最小化装饰元素
- 专注内容本身

### Key Design Elements

1. **干净的阅读界面** - 单栏居中，无干扰
2. **高效的编辑体验** - 左右分栏，实时预览
3. **明确的操作反馈** - Toast提示，状态变化
